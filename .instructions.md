# Instructions de développement Velo-Info

Ce projet utilise **Angular 21+ avec des conventions modernes strictes**.

## ⚠️ RÈGLES CRITIQUES - TOUJOURS RESPECTER

### Angular - Patterns INTERDITS ❌

**NE JAMAIS utiliser les décorateurs dépréciés :**
- ❌ `@Input()` - Utiliser `input()` à la place
- ❌ `@Output()` - Utiliser `output()` à la place  
- ❌ `@ViewChild()` - Utiliser `viewChild()` à la place
- ❌ `@ViewChildren()` - Utiliser `viewChildren()` à la place
- ❌ `@ContentChild()` - Utiliser `contentChild()` à la place
- ❌ `@ContentChildren()` - Utiliser `contentChildren()` à la place

**Exemples corrects Angular 21+ :**

```typescript
// ✅ BON - Angular 21+ avec signals
import { Component, input, output, viewChild } from '@angular/core';

export class MyComponent {
  // Props
  title = input<string>(); // au lieu de @Input()
  count = input.required<number>(); // obligatoire
  
  // Events
  save = output<string>(); // au lieu de @Output()
  
  // Query
  button = viewChild<ElementRef>('saveBtn'); // au lieu de @ViewChild
}
```

```typescript
// ❌ MAUVAIS - Pattern déprécié
import { Component, Input, Output, EventEmitter, ViewChild } from '@angular/core';

export class MyComponent {
  @Input() title?: string;
  @Output() save = new EventEmitter<string>();
  @ViewChild('saveBtn') button?: ElementRef;
}
```

### CSS - Patterns INTERDITS ❌

- ❌ `::ng-deep` - Obsolète et déprécié
- ❌ `!important` - Sauf cas exceptionnel pour écraser du JS
- ❌ Sélecteurs imbriqués profonds - Privilégier BEM
- ❌ IDs pour le style - Toujours utiliser des classes
- ❌ Media queries répétées - Un seul sélecteur avec `@include`

## 🏗️ Architecture du projet

### Structure Angular
- **Standalone components** uniquement (pas de NgModules)
- **Signals** pour le state management (`signal()`, `computed()`, `effect()`)
- **SignalStore (NGRX)** pour les stores complexes
- **Typed forms** avec les nouvelles API Angular 21

### Structure CSS (ITCSS)
```
/src/css/
  ├── /settings    # Variables, couleurs
  ├── /tools       # Mixins, fonctions
  ├── /generic     # Reset, box-sizing
  ├── /elements    # Éléments HTML de base
  ├── /components  # Composants réutilisables
  └── /utils       # Classes utilitaires
```

### Naming conventions
- Components : `feature.ts` (pas `feature.component.ts`)
- Services : `feature.service.ts`
- BEM pour CSS : `.block__element--modifier`

## 🔥 Firebase/Firestore

- Collection `users/{userId}/activities/{activityId}`
- Collection `users/{userId}/stats_month/{year-month}`
- Collection `users/{userId}/stats_global/global`
- **Important** : Toujours recalculer les stats au lieu d'incrémenter

## 🚴 Spécificités métier

- Filtrer uniquement les activités `type === 'Ride'`
- Utiliser `summary_polyline` de Strava (déjà disponible)
- Topologie embarquée dans les segments
- Rate limit Strava : 200 appels / 15 minutes
